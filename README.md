# Ask the Brand — Acme Co. RAG

A small Retrieval-Augmented Generation app over a synthetic corpus of ~1,500 social
posts and news snippets about a **fictional** brand, *Acme Co.* You can ask grounded
questions, summarize over metadata filters, and compare how the conversation changed
before vs. after an event. Answers are **grounded**, **cited**, and the system
**refuses** when the corpus doesn't support an answer.

Built with **LangChain** + **OpenAI** (`gpt-4o-mini`, `text-embedding-3-small`),
**Weaviate** (local, hybrid search), a **FastAPI** API, and a **Streamlit** UI.

```
   corpus.jsonl ─────► │  ingest.py   │  chunk → OpenAI embed → upsert
                       └──────┬───────┘            (incremental via manifest)
                              ▼
   Streamlit UI ──HTTP──►  FastAPI  ──►  retrieval.py  ──►  Weaviate (hybrid: BM25 + vector)

```

## Quickstart

**Prerequisites:** Python 3.11/3.12 (3.12 recommended), Docker, an OpenAI API key.

```bash
# 1) Config — add your OpenAI key
cp .env.example .env
# edit .env and set OPENAI_API_KEY=sk-...

# 2) Python env
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 3) Start Weaviate (local, via Docker)
docker compose up -d        # waits ~10s to be ready

# 4) Generate the corpus (LLM-generated; ~1500 docs).
#    A pre-generated data/corpus.jsonl is committed, so you can skip this.
python scripts/generate_corpus.py            # add --offline for no-API template bodies

# 5) Ingest (chunk + embed + index)
export PYTHONPATH=src
python scripts/run_ingest.py --path data/corpus.jsonl

# 6) Run the API + UI (two terminals)
python -m uvicorn ask_the_brand.api:app --port 8000        # terminal A  (PYTHONPATH=src)
python -m streamlit run app/streamlit_app.py               # terminal B  → http://localhost:8501
```

> A `Makefile` wraps these: `make weaviate-up`, `make corpus`, `make ingest`, `make api`, `make ui`, `make eval`.

### Try it (API)

```bash
# Grounded answer with citations
curl -s localhost:8000/ask -H 'content-type: application/json' \
  -d '{"question":"How long does the Rocket Skates battery last?"}' | jq

# Refusal (out of corpus)
curl -s localhost:8000/ask -H 'content-type: application/json' \
  -d '{"question":"What is Acme'\''s stock price today?"}' | jq '.refused, .answer'

# Metadata filter changes what is retrieved
curl -s localhost:8000/search -H 'content-type: application/json' \
  -d '{"query":"battery","filters":{"sources":["news"],"sentiments":["negative"]}}' | jq '.results[].doc_id'

# Summarize over a filter
curl -s localhost:8000/summarize -H 'content-type: application/json' \
  -d '{"instruction":"Summarize the negative news about the Rocket Skates.","filters":{"sources":["news"],"sentiments":["negative"]}}' | jq

# Compare sentiment before/after the April 10 launch
curl -s localhost:8000/compare -H 'content-type: application/json' \
  -d '{"topic":"the Acme Rocket Skates","split_date":"2026-04-10"}' | jq
```

## The corpus

Generated by [corpus_generator.py](src/ask_the_brand/corpus_generator.py). A **deterministic,
seeded plan** assigns each doc a theme, source, author, sentiment, and date; the **LLM
writes only the `body`** (batched for cost) so the mix is reproducible and cheap. Use
`--offline` to fill bodies from templates with no API calls.

- **Themes:** Rocket Skates launch (2026-04-10), customer support, pricing/Acme Prime,
  sustainability, a safety-recall controversy, battery/performance.
- **Sources:** `twitter`, `reddit`, `forum` (social) + `news`, `blog`.
- **Sentiment:** mix of `positive` / `negative` / `neutral`, skewed per theme.
- **Anchor facts** (stable ids like `acme-anchor-launch`) give the eval reliable ground truth.
- **Deliberate contradictions** (stable ids `acme-contra-*`):
  - battery life **20 hours** vs **dies in ~3 hours**
  - Acme **issues a recall** vs **denies any recall**
  - Acme Prime price **doubled** vs **unchanged at $9.99**

`data/new_docs.jsonl` holds ~25 June-dated docs (incl. a "Rocket Skates v2" announcement)
used to demonstrate **incremental ingestion**.

## Design choices (for the walkthrough)

- **Chunking** — `RecursiveCharacterTextSplitter`, `chunk_size=512`, `overlap=64`. The docs
  are short (mostly social posts), so most become a single chunk; chunking still matters for
  the longer news/blog snippets and keeps the pipeline honest. Each chunk carries the parent
  doc's metadata + `chunk_index`.
- **Embeddings** — OpenAI `text-embedding-3-small` (1536-d): cheap, strong, fast. We bring
  our own vectors; Weaviate is configured with `vectorizer=none`.
- **Hybrid retrieval & the merge** — Weaviate runs BM25 (keyword) and vector (dense) search
  separately, then fuses with **`relativeScoreFusion`**: each result set's scores are
  min-max normalized to [0,1] and combined as `alpha * vector + (1 - alpha) * bm25`.
  `alpha=1` → pure vector, `alpha=0` → pure BM25, default **0.5**. One hybrid call gets both
  signals; `alpha` is exposed in the API and UI so you can watch ranking shift.
- **Metadata filters** — translated to Weaviate `Filter`s and ANDed: `source` / `sentiment`
  via OR-of-equals, `posted_at` via a date range. Applied *inside* the hybrid query so they
  genuinely constrain candidates (not post-filtering).
- **top_k** — default **5** for Q&A (precision), larger (~40) for summarize/compare (recall
  across many short docs).
- **Prompt / grounding / citations** — the model is instructed to answer *only* from the
  tagged `[doc_id]` context, cite every id it uses, and flag contradictions. Citations in the
  response are the cited ids mapped back to retrieved chunks (invented ids are dropped).
- **Refusal** — two gates: (1) a pre-LLM score gate — if no chunk clears
  `REFUSAL_SCORE_THRESHOLD`, we refuse without calling the LLM; (2) the model emits
  `INSUFFICIENT_CONTEXT` when the retrieved context doesn't support an answer.
- **Incremental ingestion** — a `data/.ingest_manifest.json` maps `doc_id → sha256(body)`.
  Only new/changed docs are (re)embedded; chunk UUIDs are deterministic (`uuid5`) so
  re-ingest upserts instead of duplicating. A second run reports `new=0 changed=0`.

## Endpoints

| Method | Path | Purpose |
|---|---|---|
| GET | `/health` | Weaviate connectivity + indexed object count |
| POST | `/ingest` | Incremental ingest of a JSONL file |
| POST | `/search` | Raw hybrid retrieval (no LLM) — shows filters/alpha changing results |
| POST | `/ask` | Grounded Q&A with citations + refusal |
| POST | `/summarize` | Grounded summarization over a metadata filter |
| POST | `/compare` | Before/after comparison around a split date |

Interactive docs at `http://localhost:8000/docs`.

## Eval

```bash
export PYTHONPATH=src
python scripts/run_eval.py
```

[eval/eval_set.json](eval/eval_set.json) has 14 hand-written items (normal Q&A, filtered
queries, the contradictions, and out-of-corpus questions that should refuse). The script
prints a per-item table plus aggregate **retrieval hit-rate**, **citation accuracy**, and
**refusal accuracy**.

## Notes

- The corpus is about a **fictional** brand; bodies are model-generated for realism.
- `text-embedding-3-small` + `gpt-4o-mini` keep costs low (full corpus generation is well
  under $0.10). Change the chat model anytime via `OPENAI_CHAT_MODEL` in `.env`.
- Python 3.12 is recommended; if a dependency lacks wheels on 3.13/3.14, use a 3.12 venv.
